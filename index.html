Why RalphTerm

The terminal is the stable contract.

AI coding tools are becoming interactive terminal products. RalphTerm embraces that instead of fighting it.

CORE PRINCIPLE

No prompt-mode dependency

Drive the official CLI through a PTY. The same interface humans use becomes the automation surface.

02

No credential copying

The vendor CLI owns login, identity, rate limits, and safety prompts. RalphTerm only hosts the terminal.

03

Built for long runs

Status, input, resize, cancel, WebSocket events, transcripts, and workflow signals are first-class.

How it works

A control plane around real terminals.

Each session gets its own PTY. RalphTerm writes input, reads output, emits structured events, and keeps an audit trail.

1
SpawnLaunch Claude Code or Codex in a dedicated PTY.
2
DrivePaste prompts and follow-up input exactly like a user.
3
ObserveStream terminal output and detect workflow markers.
4
AuditPersist transcripts, events, decisions, and final results.
