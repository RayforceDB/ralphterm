# crates.io publish + install.sh ‚Äî Design Spec

**Date:** 2026-05-17
**Goal:** Make RalphTerm installable via two standard channels: `cargo install ralphterm` (single crate, both binaries) and `curl -sSf https://ralphterm.rayforcedb.com/install.sh | sh` (cross-platform prebuilt-binary installer, generated by `cargo-dist`).

## Decisions captured during brainstorming

| Decision | Choice |
|---|---|
| crates.io scope | Single crate `ralphterm`, both binaries shipped inside it |
| Release tooling | `cargo-dist` (purpose-built for Rust binary distribution) |
| Install path | `$HOME/.local/bin` on Unix; `%LOCALAPPDATA%\Programs\ralphterm` on Windows |
| Platforms | linux-x64, linux-arm64, darwin-x64, darwin-arm64, windows-x64 |
| Source-build fallback | None ‚Äî prebuilt only. Users without a matching prebuilt fall back to `cargo install ralphterm`. |

---

## Crate hygiene

`src/lib.rs` currently re-exports every internal module (`cli`, `runner`, `runs`, `signals`, `store`, `pty_agent`, `workspace`, `plan`, `config`, `notify`, `docker`). The lib exists only so both `src/main.rs` and `src/bin/ralphex.rs` can call `ralphterm::cli::run()`. We are NOT promising any of those modules as stable public API.

Changes:

- Add `#[doc(hidden)]` to every `pub mod` in `src/lib.rs`. This keeps the symbols reachable from both binaries but hides them from rustdoc/docs.rs.
- Add a single doc comment at the top of `src/lib.rs`:
  ```rust
  //! Internal library for the `ralphterm` and `ralphex` binaries.
  //!
  //! No items in this crate are part of a stable public API. The library
  //! exists only to let the two binaries share the CLI entry point. If you
  //! depend on this crate as a library, expect breaking changes between
  //! every release.
  ```

No file split, no workspace, no API redesign. The decisions log already ruled out a workspace.

## `Cargo.toml` changes

Add to `[package]`:

```toml
description = "Drop-in replacement for ralphex. Runs Claude Code / Codex / Gemini in real PTYs with independent review gates."
authors = ["Hetoku <volonter84@gmail.com>"]
repository = "https://github.com/RayforceDB/ralphterm"
homepage = "https://ralphterm.rayforcedb.com"
documentation = "https://ralphterm.rayforcedb.com/docs/"
readme = "README.md"
keywords = ["ralphex", "claude", "codex", "pty", "plan-runner"]
categories = ["command-line-utilities", "development-tools"]
rust-version = "1.75"
include = [
    "src/**/*",
    "scripts/wrappers/**/*",
    "Cargo.toml",
    "Cargo.lock",
    "README.md",
    "LICENSE",
]
```

`include` is preferred over `exclude` because crates.io packaging is opt-in ‚Äî we list exactly what ships. The wrapper scripts are included so `cargo install`'d binaries can locate them via the `scripts/wrappers/` dev fallback (see `src/config.rs::locate_wrapper_script`).

Add explicit `[[bin]]` entries (currently both binaries are auto-discovered, but being explicit prevents surprises on crates.io):

```toml
[[bin]]
name = "ralphterm"
path = "src/main.rs"

[[bin]]
name = "ralphex"
path = "src/bin/ralphex.rs"
```

The `description` field already exists; the new text replaces the existing one. The current description ("Terminal-native orchestration API for interactive Claude/Codex agent workflows") is accurate but doesn't lead with the drop-in story.

## cargo-dist setup

`cargo install cargo-dist@0.22` (pin a known-working version), then `cargo dist init --yes` with the following config saved to `dist-workspace.toml`:

```toml
[dist]
cargo-dist-version = "0.22.0"
ci = "github"
installers = ["shell", "powershell"]
targets = [
    "aarch64-apple-darwin",
    "x86_64-apple-darwin",
    "aarch64-unknown-linux-gnu",
    "x86_64-unknown-linux-gnu",
    "x86_64-pc-windows-msvc",
]
install-path = "$HOME/.local/bin"
install-updater = false
pr-run-mode = "plan"
allow-dirty = ["ci"]
```

cargo-dist generates these files (we commit them as-is):

- `.github/workflows/release.yml` ‚Äî the release pipeline.
- `dist-workspace.toml` ‚Äî the configuration above.

It also generates artifacts at release time (uploaded to the GitHub release, not committed):

- `ralphterm-installer.sh` ‚Äî Unix installer.
- `ralphterm-installer.ps1` ‚Äî Windows PowerShell installer.
- One `ralphterm-aarch64-apple-darwin.tar.xz`-style tarball per target.
- A `dist-manifest.json` summarising the release.
- A `SHA256SUMS` file with checksums for each archive.

`pr-run-mode = "plan"` means PRs run `cargo dist plan` (a dry validation) instead of full builds ‚Äî fast CI on PRs, real builds only on tag pushes.

`install-updater = false` skips bundling a self-updater into the binaries. We can flip this later.

## Release flow

1. **Pre-flight on `main`** (one-time, this slice):
   - Land all the `Cargo.toml` + `lib.rs` + cargo-dist changes.
   - Run `cargo publish --dry-run` locally and resolve any packaging issues (missing files, invalid metadata).
2. **Cut a release tag:**
   - `git tag v0.1.0 -m "v0.1.0 ‚Äî first crates.io release"`.
   - `git push origin v0.1.0`.
3. **GH Actions release workflow fires:**
   - Builds the five platform binaries in parallel.
   - Builds the two installer scripts (`.sh`, `.ps1`).
   - Creates a GitHub Release named `v0.1.0` with all the artifacts attached.
4. **Publish to crates.io** (manual, run from a clean main checkout after the tag):
   - `cargo publish` (assuming `cargo login` has been done once).
   - We do NOT automate publish from CI in this slice ‚Äî manual publish keeps the first release deliberate. A follow-up can wire `cargo dist plan --auto-release` once we're confident.

## install.sh delivery

Two URLs, one underlying artifact:

- **Canonical** (always works, no DNS dependence):
  `https://github.com/RayforceDB/ralphterm/releases/latest/download/ralphterm-installer.sh`
  This is what `cargo-dist` publishes. The README + docs point at this URL.
- **Vanity** (short, brandable):
  `https://ralphterm.rayforcedb.com/install.sh`
  A static file in `site/install.sh` that `curl`s the canonical URL and pipes to `sh`. Deployed via the existing `pages.yml` workflow.

`site/install.sh` content:

```sh
#!/bin/sh
# RalphTerm one-line installer.
# Downloads and runs the cargo-dist-generated installer from the latest GitHub release.
set -eu
exec sh -c "$(curl -fsSL https://github.com/RayforceDB/ralphterm/releases/latest/download/ralphterm-installer.sh)" -- "$@"
```

`site/install.ps1`:

```powershell
# RalphTerm one-line installer (Windows PowerShell).
# Downloads and runs the cargo-dist-generated installer from the latest GitHub release.
$ErrorActionPreference = 'Stop'
$installerUrl = 'https://github.com/RayforceDB/ralphterm/releases/latest/download/ralphterm-installer.ps1'
iex (irm $installerUrl)
```

Both shims forward args (`exec sh -c ... -- "$@"`) so cargo-dist's `--help` / `--version` / `--no-modify-path` flags still work end-to-end.

## Testing

### Cargo packaging dry-run in CI

Add a step to `.github/workflows/ci.yml`:

```yaml
- name: Cargo publish dry-run
  run: cargo publish --dry-run --allow-dirty
```

Runs on every PR. Catches missing files, invalid metadata, license issues, dependency version problems before they hit a release tag.

### install.sh smoke test

New file `tests/install_script.rs`:

```rust
#[test]
fn vanity_install_sh_redirects_to_cargo_dist_installer() {
    let body = std::fs::read_to_string("site/install.sh").expect("read site/install.sh");
    assert!(body.starts_with("#!/bin/sh\n"), "install.sh must start with #!/bin/sh");
    assert!(body.contains("set -eu"), "install.sh must set -eu");
    assert!(
        body.contains("github.com/RayforceDB/ralphterm/releases/latest/download/ralphterm-installer.sh"),
        "install.sh must redirect to the cargo-dist installer URL"
    );
    assert!(body.contains("exec sh -c"), "install.sh must exec the downloaded installer");
    assert!(body.contains("\"$@\""), "install.sh must forward args to the downstream installer");
}

#[test]
fn vanity_install_ps1_redirects_to_cargo_dist_installer() {
    let body = std::fs::read_to_string("site/install.ps1").expect("read site/install.ps1");
    assert!(body.contains("ErrorActionPreference = 'Stop'"));
    assert!(
        body.contains("github.com/RayforceDB/ralphterm/releases/latest/download/ralphterm-installer.ps1"),
        "install.ps1 must redirect to the cargo-dist installer URL"
    );
    assert!(body.contains("iex (irm $installerUrl)"));
}

#[test]
fn cargo_toml_has_publish_metadata() {
    let cargo = std::fs::read_to_string("Cargo.toml").expect("read Cargo.toml");
    for field in &["description = ", "license = ", "repository = ", "homepage = ", "readme = ", "keywords = ", "categories = "] {
        assert!(cargo.contains(field), "Cargo.toml must declare {field}");
    }
    assert!(cargo.contains("\"ralphex\""), "Cargo.toml keywords must include \"ralphex\"");
}
```

### Manual verification (one-time, before publishing)

1. Run `cargo publish --dry-run` and inspect the packaged file list. Confirm `scripts/wrappers/*.sh` is included; confirm `site/`, `docs/`, `tests/`, `target/`, `dashboard/`, `.superpowers/` are NOT.
2. After tagging `v0.1.0`, watch the release workflow run to green. Inspect the artifacts on the GH Release page.
3. Download the linux-x64 tarball, extract it, run `./ralphterm --help` and `./ralphex --help`. Both must work.
4. On a throwaway VM: `curl -sSf https://ralphterm.rayforcedb.com/install.sh | sh`. Verify both binaries land in `~/.local/bin/` and run. (After site deploy.)

## Out of scope (deferred follow-ups)

- Homebrew tap publishing. `cargo-dist` supports it via the `tap` config key; defer until v0.2 when we own a `homebrew-ralphterm` tap repo.
- AUR PKGBUILD.
- Pushing the `docker/Dockerfile` image to `ghcr.io/rayforcedb/ralphterm` as a release artifact.
- A `ralphex` crates.io name squat (we publish as `ralphterm`; the `ralphex` binary is bundled).
- `ralphterm self-update` subcommand (cargo-dist supports an updater; we set `install-updater = false` for now).
- Automating `cargo publish` from CI (manual publish keeps the first release deliberate).
- Auto-generating a `CHANGELOG.md` from conventional commits (cargo-dist can do it; defer).

## Acceptance

The slice is done when:

1. `Cargo.toml` carries all required `[package]` metadata; `cargo publish --dry-run` passes on a clean main checkout.
2. `src/lib.rs` marks every `pub mod` `#[doc(hidden)]` and explains the internal-only contract in its module doc.
3. `dist-workspace.toml` exists; `.github/workflows/release.yml` exists; both are committed as cargo-dist generated them.
4. `site/install.sh` + `site/install.ps1` exist and pass `tests/install_script.rs` (3 tests).
5. `README.md` + `site/index.html` advertise the three install commands.
6. CI workflow runs `cargo publish --dry-run` on every PR.
7. `cargo fmt --all -- --check`, `cargo clippy --all-targets --all-features -- -D warnings`, `cargo test --all` all pass.
8. (Manual, post-merge) Tagging `v0.1.0` triggers a successful release build; `cargo publish` pushes `ralphterm v0.1.0` to crates.io.
