# crates.io publish + install.sh Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship `ralphterm` on crates.io as a single crate containing both `ralphterm` and `ralphex` binaries, plus a cargo-dist-generated release pipeline whose installers (`install.sh`, `install.ps1`) are reachable via short vanity URLs on the website.

**Architecture:** Six tightly-scoped changes — Cargo metadata + lib hygiene, cargo-dist init, vanity installer shims, README/site install block, CI publish dry-run gate, integration tests. The tag-and-publish step is a one-time manual action documented at the end; the plan stops at "release-ready" so the engineer can choose when to cut v0.1.0.

**Tech Stack:** Rust 2021, `cargo-dist` 0.22, GitHub Actions, POSIX sh + PowerShell installer shims, static HTML.

#[test]
fn vanity_install_sh_redirects_to_cargo_dist_installer() {
    let body = std::fs::read_to_string("site/install.sh").expect("read site/install.sh");
    assert!(
        body.starts_with("#!/bin/sh\n"),
        "install.sh must start with #!/bin/sh"
    );
    assert!(body.contains("set -eu"), "install.sh must `set -eu`");
    assert!(
        body.contains(
            "github.com/RayforceDB/ralphterm/releases/latest/download/ralphterm-installer.sh"
        ),
        "install.sh must redirect to the cargo-dist installer URL"
    );
    assert!(
        body.contains("exec sh -c"),
        "install.sh must exec the downloaded installer"
    );
    assert!(
        body.contains("\"$@\""),
        "install.sh must forward args to the downstream installer"
    );
}

#[test]
fn vanity_install_ps1_redirects_to_cargo_dist_installer() {
    let body = std::fs::read_to_string("site/install.ps1").expect("read site/install.ps1");
    assert!(
        body.contains("ErrorActionPreference = 'Stop'"),
        "install.ps1 must set ErrorActionPreference to Stop"
    );
    assert!(
        body.contains(
            "github.com/RayforceDB/ralphterm/releases/latest/download/ralphterm-installer.ps1"
        ),
        "install.ps1 must redirect to the cargo-dist installer URL"
    );
    assert!(
        body.contains("iex (irm $installerUrl)"),
        "install.ps1 must invoke the downloaded installer via iex (irm ...)"
    );
}

#[test]
fn cargo_toml_has_publish_metadata() {
    let cargo = std::fs::read_to_string("Cargo.toml").expect("read Cargo.toml");
    for field in &[
        "description = ",
        "license = ",
        "repository = ",
        "homepage = ",
        "readme = ",
        "keywords = ",
        "categories = ",
        "authors = ",
        "documentation = ",
    ] {
        assert!(
            cargo.contains(field),
            "Cargo.toml must declare {field}"
        );
    }
    assert!(
        cargo.contains("\"ralphex\""),
        "Cargo.toml keywords must include \"ralphex\""
    );
    assert!(
        cargo.contains("[[bin]]\nname = \"ralphterm\"")
            || cargo.contains("[[bin]]\r\nname = \"ralphterm\""),
        "Cargo.toml must declare the ralphterm binary"
    );
    assert!(
        cargo.contains("[[bin]]\nname = \"ralphex\"")
            || cargo.contains("[[bin]]\r\nname = \"ralphex\""),
        "Cargo.toml must declare the ralphex binary"
    );
}

## Post-implementation manual release (not committed by the plan)

After all 8 tasks land and CI is green, the release flow is a manual sequence the engineer runs ONE TIME to cut v0.1.0. These are not committed to the repo — they're documented here so the engineer knows what to do next.

```bash
# 1. Cut the tag.
git tag v0.1.0 -m "v0.1.0 — first crates.io release"
git push origin v0.1.0

# 2. Watch the release workflow.
gh run watch --workflow=release.yml
# Or: open https://github.com/RayforceDB/ralphterm/actions

# 3. Inspect the published GitHub Release artifacts.
gh release view v0.1.0

# 4. Publish to crates.io (requires `cargo login` having been done once).
cargo publish

# 5. Verify.
cargo search ralphterm
curl -sI https://github.com/RayforceDB/ralphterm/releases/latest/download/ralphterm-installer.sh | head -3
curl -sSf https://ralphterm.rayforcedb.com/install.sh | head -5
```

If any step fails:
- The release workflow failing means the cargo-dist YAML or a platform build broke. Inspect the GH Actions logs.
- `cargo publish` rejecting with "crate name ralphterm is already taken" means someone squatted while we worked — pick an alternate name (`ralphterm-cli` is a safe fallback) and bump it in Cargo.toml.
- The vanity URL 404ing means the GitHub Pages deploy hasn't picked up the new `site/install.sh` yet. Wait ~5 minutes and retry.
