Drop-in ralphex replacement

Drop-in ralphex replacement. Real PTY, real CLIs, real review gates.

A drop-in replacement for ralphex. Swap ralphex for ralphterm, or use the bundled ralphex binary alias, and keep the same plans, prompts, config, agents, and notifications. RalphTerm reads ~/.config/ralphex/ and .ralphex/, accepts the same flags, runs the same plan format, and preserves the independent-review acceptance gate.

Write a plan. Let real terminal agents execute it, then review it. RalphTerm runs the maintainer loop directly: implementation agent, validation commands, independent reviewer, then accepted progress. It runs the official CLIs in real PTYs, sends each task as terminal input, waits for completion, and keeps the transcript. Use it for autonomous engineering loops that need real interaction without claude -p, --print, or brittle one-shot prompt calls. Unchecked agent progress is not good enough.

What is RalphTerm?

A drop-in plan runner with real review gates.

RalphTerm does not replace Claude Code or Codex. It runs them the way a human runs them: inside a terminal. The product layer is the verified plan loop around that terminal: task prompts, validation, independent review, checked-off progress, commits, and transcripts. If you are coming from ralphex, the swap is one binary; everything else stays.

What's different

Same surface. Different floor.

Same flags. Same plans. Same config.

RalphTerm accepts the same CLI surface as ralphex, reads ~/.config/ralphex/ and .ralphex/, and runs the exact same markdown plan files. Point your scripts at ralphex (the alias) or ralphterm and keep going.

Real PTY execution.

Every implementation and review command runs in a real terminal. No --print, no -p, no one-shot prompt mode. Login screens, approval prompts, rate limits, and follow-up questions all survive.

Independent review gates.

The reviewer runs in its own PTY after validation and must print an exact REVIEW_PASS line. Implementation completion alone never marks a task [x] or creates a commit.

Drop-in features

Everything ralphex did, in one binary.

Worktrees

--worktree --branch creates isolated git worktrees per plan run, defaulting the branch to the plan filename slug. Safe to run untrusted plans without touching your checkout.

Notifications

Telegram, Slack, Email (SMTP), and generic Webhook delivery on plan completion, task failure, review failure, and rate-limit pauses. Configured via [notify] or CLI overrides. See notifications.

Docker isolation

--docker --docker-image ralphterm:latest runs the agent and reviewer inside a container. Honors RALPHEX_EXTRA_VOLUMES and RALPHEX_EXTRA_ENV. See docker.

Multi-provider

Wrapper scripts for Codex, Copilot, Gemini, and OpenCode ship in scripts/wrappers/. Pick one with --agent codex, --agent copilot, --agent gemini, or --agent opencode. See providers.

Review retry loop with patience

--review-patience N and --max-iterations N let the implementation/review loop re-run with reviewer feedback before blocking. REVIEW_FAIL triggers retry; only a clean review accepts the task.

Plan-completion auto-move

On successful full or tasks mode runs, RalphTerm honors move_plan_on_completion and moves the plan to docs/plans/completed/ so the next pass sees a clean queue.

Current workflow

Plan runner first. PTY API underneath.

Use ralphterm [flags] PLAN (or the ralphex alias) for ralphex-style plan execution. The session API remains the lower-level control surface for custom orchestrators.

  • ralphterm --tasks-only docs/plans/foo.mdtasks-only mode, ralphex-style, no review gate
  • ralphterm --claude-command claude docs/plans/foo.mdfull mode with default review gate, drives the CLI in a real PTY
  • ralphterm --external-review-tool=custom --custom-review-script ./review.sh docs/plans/foo.mdmap a custom reviewer into RalphTerm's independent review command
  • ralphterm run PLAN --dry-runpreview pending tasks, review mode, and validation without starting an agent
  • ralphterm run PLAN --agent claude --require-review --review-agent codexfail before agent execution unless an independent reviewer is configured
  • POST /v1/sessionsstart Claude, Codex, or a custom command in a PTY
  • GET /v1/sessions/:id/transcriptread the terminal transcript

Get started

Swap one binary. Keep everything else.

The drop-in path is short: install RalphTerm, point your existing scripts at ralphex (the alias) or ralphterm, keep your ~/.config/ralphex/ and project .ralphex/, and confirm the review gate with ralphterm --tasks-only <plan> followed by a full reviewed run.
