RALPHTERM / v0.3.0 / MIT

A programmable PTY for the official AI CLIs.

Drop-in for ralphex 1.2.0. Same plan, same flags, same 3-phase review pipeline — but driven through a real PTY against the interactive claude and codex CLIs. The official tools own login, model selection, safety prompts, and rate limits; ralphterm owns the loop, transcripts, signals, review gates, and notifications.

— drop in

$ ralphex --tasks-only docs/plans/feature.md
≡ 
$ ralphterm --tasks-only docs/plans/feature.md

Same flags, same plan, same output. The ralphex binary is an alias for ralphterm — point your scripts at either.

— what it replaces

Prompt mode breaks. Long plan runs hit login screens, approval prompts, rate limits, follow-up questions, and output changes.

PTY mode survives. The official CLI keeps its real terminal. RalphTerm watches output and sends input like a controlled operator.

Runs stay auditable. Every task exposes validation results, completion signals, commit hashes, and the raw transcript.

— invoke

$ ralphterm --tasks-only docs/plans/feature.md

Tasks-only mode skips the independent review gate. For full mode add --external-review-tool=custom --custom-review-script <cmd>.

— more

Worktrees

Isolated git worktrees per plan; auto-derived branch slug from the plan filename.

Notifications

Telegram, Slack, Email (SMTP), and Webhook deliveries on plan / task / review / rate-limit events.

Docker

Optional containerized execution. Honors RALPHEX_EXTRA_VOLUMES + EXTRA_ENV.

Multi-provider

Bundled wrappers for Codex, Copilot, Gemini, and OpenCode CLIs.

Review retry

Implementer + reviewer fixer loop with patience / stalemate detection.

Plan auto-move

Successful plans relocate to docs/plans/completed/ on configurable opt-in.
