For plan-driven Claude Code and Codex runs

Write a plan. Let real terminal agents execute it.

RalphTerm runs markdown plans through official AI CLIs in real PTYs. It sends each task as terminal input, waits for completion, runs validation commands, marks finished checkboxes, commits progress, and stores transcripts.

Use it for autonomous engineering loops that need real interaction without claude -p, --print, or brittle one-shot prompt calls.

What is RalphTerm?

A plan runner for autonomous coding work.

RalphTerm does not replace Claude Code or Codex. It runs them the way a human runs them: inside a terminal. The product layer is the plan loop around that terminal: task prompts, validation, checked-off progress, commits, and transcripts.

Why it exists

AI CLIs are interactive. Automation has to handle that.

Prompt mode breaks

Long plan runs hit login screens, approval prompts, rate limits, follow-up questions, and output changes.

PTY mode survives

The official CLI keeps its real terminal. RalphTerm watches output and sends input like a controlled operator.

Runs stay auditable

Every task can expose validation results, completion signals, commit hashes, and the raw transcript.

Current workflow

Plan runner first. PTY API underneath.

Use ralphterm run for task-by-task plan execution. The session API remains the lower-level control surface for custom orchestrators.

  • ralphterm smoke --agent claudeverify the real CLI can run inside a PTY and emit a completion signal
  • ralphterm run PLAN --dry-runpreview pending tasks and validation without starting an agent
  • ralphterm run PLAN --agent claude --require-review --review-command "codex exec review-task"fail before agent execution unless an independent reviewer command is configured
  • ralphterm run PLAN --agent codexrun the same loop through Codex
  • POST /v1/sessionsstart Claude, Codex, or a custom command in a PTY
  • GET /v1/sessions/:id/transcriptread the terminal transcript
