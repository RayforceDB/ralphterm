RALPHTERM / v0.3.11 / MIT

Long AI coding sessions, unattended.

The official claude and codex CLIs are interactive REPLs built for one prompt at a time. Real work isn't one prompt — it's read the plan, pick the next task, edit files, run validation, commit, mark done, do the next one, for hours. RalphTerm wraps the interactive CLI in a controlled PTY and runs that loop for you. Per-task commits. Parallel review gate before merge. Transcripts of every iteration. A web dashboard to watch from.

— how it works

Spawn the real CLI in a PTY. Same binary a human would type. Same login, same model, same safety prompts, same rate limits — those still belong to the upstream tool.

Drive the plan loop. Read the next unchecked checkbox, paste it as the prompt, capture the response via a file-handoff side channel, commit, mark done, repeat. Per-iteration transcripts written to .ralphex/progress/.

Gate the merge. Optional 3-phase parallel review pipeline (5 reviewers → external/codex fixer loop → 2 reviewers). Critical findings block the merge; the implementer gets a retry budget.

Watch from a dashboard. ralphterm serve exposes a local HTTP API + WebSocket transcripts. Notifications go out over Telegram, Slack, Email, or Webhook.

— invoke

$ ralphterm --tasks-only docs/plans/feature.md

Tasks-only mode skips the independent review gate. For full mode add --external-review-tool=custom --custom-review-script <cmd>.

— install

$ curl -sSf https://ralphterm.rayforcedb.com/install.sh | sh
PS> irm https://ralphterm.rayforcedb.com/install.ps1 | iex

The installer lands ralphterm in ~/.local/bin and supports ralphterm update for in-place upgrades.

— more

Worktrees

Isolated git worktrees per plan; auto-derived branch slug from the plan filename.

Notifications

Telegram, Slack, Email (SMTP), and Webhook deliveries on plan / task / review / rate-limit events.

Docker

Optional containerized execution. Honors RALPHEX_EXTRA_VOLUMES + EXTRA_ENV.

Multi-provider

Bundled wrappers for Codex, Copilot, Gemini, and OpenCode CLIs.

Review retry

Implementer + reviewer fixer loop with patience / stalemate detection.

Plan auto-move

Successful plans relocate to docs/plans/completed/ on configurable opt-in.
