The problem

One-shot prompt mode was never enough for autonomous coding.

It is tempting to run an AI CLI with a prompt flag and parse the text that comes back. That works for demos. It breaks when the CLI asks for login, asks to approve a file edit, changes output format, opens an interactive flow, or needs follow-up input. RalphTerm is built for that reality.

CORE PRINCIPLE

No dependency on prompt mode

The official CLI runs inside a PTY. Your orchestrator talks to the session, not to a brittle one-shot wrapper.

SAFETY

Interactive behavior is expected

Login, rate-limit notices, approval prompts, and tool confirmations are part of the flow. RalphTerm streams and records them instead of pretending they do not exist.

CONTROL

Built for long agent runs

Create a session, send input, resize it, cancel it, read status, stream events, and keep the transcript.

What it gives you

A drop-in control plane for CLI-based agents.

RalphTerm starts by replacing fragile prompt-mode execution. Then it adds the missing pieces for real autonomous work: planning, workspaces, reviews, approvals, transcripts, and final patches.

Session API

Start Claude Code, Codex, or another command as an API-controlled PTY session. Keep using the official CLI.

Completion signals

Detect COMPLETED, FAILED, QUESTION, PLAN_READY, and REVIEW_DONE from real terminal output.

Approval handling

Approvals stay visible. You can pause, notify a human, or apply explicit policy. No hidden bypass.

Audit trail

Every run can produce live output, final artifacts, transcripts, and event history.

Milestone 1

Milestone 1: replace prompt-mode agent runners completely.
