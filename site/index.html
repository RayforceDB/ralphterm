Why RalphTerm

The terminal is the stable contract.

AI coding products move fast. Their terminal interface remains the one surface users trust. RalphTerm turns that surface into infrastructure without pretending to be the provider.

CORE PRINCIPLE

No prompt-mode dependency

Drive the official CLI through a PTY. The same interface humans use becomes the automation surface.

SAFETY

Vendor auth stays vendor-owned

The CLI owns login, identity, rate limits, account policy, and safety prompts. RalphTerm hosts the terminal and records the run.

CONTROL

Built for long runs

Create, input, resize, cancel, stream, transcript, and detect workflow signals through a small local API.

Product surface

From terminal session to engineering run.

The PTY core is only the beginning. Milestone 1 turns RalphTerm into an autonomous engineering loop with planning, implementation, review, approvals, and final artifacts.

Session API

One endpoint starts an agent. Others drive input, resize the terminal, cancel work, fetch status, stream events, and pull transcripts.

Workflow signals

RalphTerm detects COMPLETED, FAILED, QUESTION, PLAN_READY, and REVIEW_DONE, then exposes them as structured events.

Approval queue

Human approval is the default. Policy-based auto-approval is explicit, auditable, and never a hidden bypass.

Dashboard-ready

Every run has phase status, live terminal output, final artifacts, transcripts, and event history.
